## 1. BUGS CORRIGIDOS NESTA SESSÃO

### Bug 1 — `generate-ideas` retornava erro 500
- **Causa:** Declaração duplicada de `AI33_API_KEY` e `OPENROUTER_API_KEY` na linha 101-102 (já declaradas na linha 15-16) — erro de compilação Deno
- **Fix:** Removidas as linhas duplicadas
- **Deploy:** ✅ ATIVO

### Bug 2 — Production Wizard: narração "pulava" e vídeo ficava sem áudio
- **Causa A:** `hub.voice === "browser"` marcava capítulos como `"browser_tts"` sem gerar blob real. `handleGenerateAllAudio` usava `if (chapter.audioUrl) continue` e pulava tudo.
- **Fix:** Quando `hub.voice === "browser"` mas `AI33_API_KEY` existe, usa AI33 TTS real. Loop agora não pula capítulos com `audioUrl === "browser_tts"`.

### Bug 3 — "Falha ao carregar imagem da cena 1"
- **Causa:** URLs externas (DALL-E/Kie.ai) bloqueadas por CORS quando canvas tenta carregar com `crossOrigin = "anonymous"`
- **Fix:** Após gerar imagem, faz `fetch()` e converte para `blob://` local antes de salvar no state. Canvas nunca vê URL externa.

### Bug 4 — Imagens continuam gerando após cancelar/sair
- **Causa:** Loop serial sem mecanismo de cancelamento
- **Fix:** `cancelImagesRef = useRef(false)` + botão ❌ vermelho que seta a ref. Loop checa antes de cada cena.

### Bug 5 — "Erro ao salvar blueprint: Cannot coerce the result to a single JSON object"
- **Causa:** `.update().single()` falha quando nenhum blueprint existe ainda (0 rows)
- **Fix:** Trocado para `.upsert({ channel_id, ...updates }, { onConflict: 'channel_id' }).maybeSingle()`

### Bug 6 — "J.map is not a function" no Studio Longo
- **Causa:** `google/gemini-2.5-flash` via OpenRouter não suporta `response_format: json_object` → retorna JSON dentro de markdown fences → `JSON.parse` falha ou retorna estrutura diferente
- **Fix:** Removido `response_format`, adicionada extração robusta com regex (strip fences + `match(/\{[\s\S]*\}/)`), validação que `scenes` é array, tratamento de modelos que encapsulam sob chave `"script"`. Deploy ✅

## 5. ANÁLISE DO PIPELINE N8N (C:\Users\Fabricio Padrin\Downloads\n8n-fluxos\n8n-fluxos)

### O que é
Pipeline de produção de vídeos 100% serverside rodando em VPS com n8n + Node.js + FFmpeg. Produz **MP4 horizontal** com qualidade profissional. 8 fluxos:

```
0. roteiro *.json       → Cron diário → Pega ideia do Supabase → OpenAI gera roteiro → salva
0. gerador de ideias    → Cron mensal → OpenAI gera ideias para 4 canais → Supabase
1. tts.json             → Cron → Google Cloud TTS Chirp3-HD + SSML → MP3 no VPS → Whisper gera SRT
2. video_ready_to_render→ Cron → OpenAI gera prompts por cena → Stability AI SDXL gera imagens → VPS
3. video_render.json    → Cron → Node.js: renderScenes.js + concatScenes.js + final-w-audio.js + add-srt.js + add-music-bg.js (tudo FFmpeg) → Google Drive
4. SEO generator        → Cron → OpenAI gera título/descrição/tags → Supabase
5. up_to_plan           → Cron → Google Sheets (calendário de conteúdo)
6. publicar to metrics  → Cron → Sheets → rastreia publicados
7. metrics to db        → Cron → YouTube API analytics → Supabase
```

### Canais suportados
- **Bíblico** — estética 2D animada, luminosa, quente, personagens bíblicos em situações simbólicas
- **Darkside (Científico)** — 3D hiper-realista, iluminação dramática, alien/surreal
- **Terror** — fotografia true crime, documental investigativo, hiper-realista
- **Jazz with J** — estilo próprio (jazz/cultura)

### Diferenciais técnicos vs AutoDark

| Feature | n8n Pipeline | AutoDark |
|---|---|---|
| **TTS** | Google Cloud Chirp3-HD pt-BR (SSML, fonética, 0.95x rate) | OpenAI TTS `alloy` |
| **SRT** | Whisper transcrevendo o áudio real (sync perfeito) | Chunks estimados por palavras |
| **Imagens** | Stability AI SDXL 1024 | DALL-E 3 / Kie.ai |
| **Motion por cena** | Baseado em emoção (urgency→zoom, shock→zoom rápido, curiosity→pan esquerda) | Ken Burns fixo |
| **Render** | FFmpeg real (MP4 H.264, audio mixing, SRT burned) | Canvas + MediaRecorder (WebM) |
| **Automação** | 100% automático | Wizard manual |

### Motion por emoção (código do pipeline — reutilizável)
```javascript
function getMotionByEmotion(emotion) {
  switch (emotion) {
    case "urgency":     return { style: "zoom", from: 1.0, to: 1.15 };
    case "curiosity":   return { style: "pan", direction: "left", intensity: 0.15 };
    case "motivation":  return { style: "zoom", from: 1.0, to: 1.08 };
    case "shock":       return { style: "zoom", from: 1.0, to: 1.2 };
    case "inspiration": return { style: "pan", direction: "right", intensity: 0.12 };
    default:            return null; // random
  }
}
```

**2a. Google Cloud TTS Chirp3-HD**
- Substituir chamada OpenAI TTS na Edge Function `youtube-generate-audio`

- Qualidade muito superior para português — diferença audível imediata
- Requer: `GOOGLE_TTS_API_KEY` nas env vars do Supabase

**2b. SRT via Whisper**
- Após gerar áudio, enviar para `https://api.openai.com/v1/audio/transcriptions`
- Gera SRT com timing real baseado no áudio
- Substitui o sistema de chunks estimados no `useVideoAssembler.tsx`

### Fase 3 — Integração com n8n VPS (longo prazo)

**Objetivo:** AutoDark gera conteúdo (scripts + imagens + áudio) → dispara webhook para o n8n VPS do usuário → n8n renderiza MP4 horizontal → sobe no Google Drive/Supabase Storage → AutoDark exibe link para download.

**Como implementar:**
1. Criar botão "Renderizar via Servidor" no Step 7 do Production Wizard
2. Criar Edge Function `trigger-render` que monta o payload e chama o webhook n8n
3. No n8n, criar um fluxo Webhook → recebe payload AutoDark → adapta para o formato existente do `video_render.json` → renderiza → chama webhook de callback
4. AutoDark polling ou Supabase realtime para mostrar progresso

**Payload que o AutoDark enviaria para o n8n:**
```json
{
  "contentId": "uuid",
  "channelId": "uuid",
  "style": "biblico|darkside|terror|jazz",
  "scenes": [
    {
      "index": 1,

      "audioUrl": "storage URL",
      "narration": "texto narrado",
      "emotion": "urgency|shock|curiosity",
      "srtStart": 0.0,
      "srtEnd": 4.5,
      "duration": 4.5

  "audioFull": "storage URL do áudio completo",
```
